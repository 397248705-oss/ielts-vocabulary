# 雅思单词本 UI 全面升级设计

## 目标

在不改变“手机浏览器使用、仅保存本地、内置雅思词库、每日默认 20 个新词、综合学习模式”这些核心约束的前提下，全面升级界面并增加错题、收藏、分类筛选、自定义单词和统计能力。

## 视觉方向

采用用户选定的“A 清透蓝白”方案：

- 明亮白色和浅灰页面背景，蓝色作为主要操作色。
- 正确、错误、收藏和自定义等状态分别使用绿、红、黄和紫作为辅助色，避免单一蓝色界面。
- 使用克制阴影、细边框和不超过 8px 的卡片圆角。
- 使用 Lucide 图标，熟悉的操作优先显示图标并提供可访问名称。
- 标题与数据层级清晰，首页信息较丰富，学习页保持沉浸和低干扰。
- 同时适配小屏手机、Pixel 7 尺寸和桌面浏览器，不使用随视口宽度缩放的字体。

## 导航与页面

底部固定五个主入口：

1. **今日**：学习目标、今日进度、连续学习天数、开始学习、复习错题、我的收藏、添加单词和学习概览。
2. **词库**：搜索、雅思主题筛选、来源筛选、收藏筛选、内置词和自定义词列表。
3. **错题**：按错误次数和最近错误时间展示，可开始专项复习或标记已掌握。
4. **统计**：总学习词数、已掌握、错题量、连续学习天数、今日完成率和近七天趋势。
5. **设置**：每日新词数、备份导出与导入、数据清理。

学习流程不占用底部独立入口，从“今日”或“错题”进入。学习页使用顶部进度条和大尺寸中心卡片，保留闪卡、选择题和拼写题三种现有模式，并重新设计作答反馈。

## 新增功能规则

### 收藏

- 所有内置词和自定义词均可收藏或取消收藏。
- 收藏不改变复习调度状态。
- 首页快捷入口和词库筛选都能查看收藏词。

### 错题本

- 选择题或拼写题答错时自动记录错误次数和最近错误时间。
- 错题专项复习沿用现有练习生成逻辑，但只从错题集合选词。
- 连续答对后可标记“已掌握”；该词不再出现在默认错题列表，但历史错误次数保留在统计中。

### 自定义单词

- 表单包含英文单词、中文释义、英文例句和中文例句；英文单词与中文释义必填。
- 重复英文单词不重复创建，界面给出明确提示。
- 保存成功后自动加入每日新词候选池，并显示在“自定义”分类。

### 筛选

- 搜索匹配英文单词或中文释义。
- 主题、来源和收藏条件可组合。
- 无结果时显示明确空状态和清除筛选入口。

## 数据与存储

- 所有新增数据继续保存在 IndexedDB，不发送到 GitHub Pages 或任何服务器。
- 在现有学习记录之外增加收藏、自定义词、错题状态和每日统计记录。
- 数据库升级使用兼容迁移，保留当前设置、复习进度和历史学习记录。
- JSON 备份包含新增数据；导入旧版本备份时为缺失字段提供安全默认值。
- 自定义词使用稳定本地 ID，避免与内置词 ID 冲突。

## 组件边界

- `App` 只负责加载本地数据、切换页面和协调学习会话。
- 独立领域函数负责收藏筛选、错题累计、错题掌握和每日统计计算。
- 独立存储函数负责数据库迁移、自定义词、收藏、错题和备份读写。
- 复用视觉组件包括页面头部、数据卡、快捷入口、筛选控件、单词行、学习进度、反馈面板和底部导航。
- 各屏幕只组合数据和组件，不直接操作 IndexedDB。

## 错误与空状态

- 数据加载失败时保留重试、导入备份和清空数据入口。
- 自定义词校验错误显示在对应字段附近，不使用浏览器弹窗。
- 备份格式无效时拒绝覆盖现有数据，并显示可理解的错误原因。
- 空错题、空收藏、空搜索结果和暂无统计均有独立空状态。
- 语音能力不可用时禁用朗读按钮，但不影响学习。

## 测试与验收

- 单元测试覆盖数据库迁移、收藏切换、组合筛选、错题累计与掌握、自定义词进入每日计划、统计计算和新旧备份恢复。
- 组件测试覆盖首页快捷入口、添加单词校验、错题操作和学习反馈。
- Playwright 覆盖手机端首页、学习、词库筛选、自定义词、错题和设置备份流程。
- 使用小屏手机、Pixel 7 和桌面尺寸截图检查文字截断、重叠、导航遮挡和视觉层级。
- 保留现有 PWA、HTTPS 和离线启动验证。
- 完成后合并到 `master`，由现有 GitHub Actions 自动发布到原网址。

## 非目标

- 不增加登录、云同步、账号系统或服务器数据库。
- 不增加多人功能、排行榜或社交分享。
- 不购买域名，也不改变现有 GitHub Pages 地址。
